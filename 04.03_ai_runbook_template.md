# AI Runbook – <PROJECT_NAME>

## 1️⃣ Overview
- **Purpose:** Describe how to **deploy**, **validate**, and **operate** the AI model `<MODEL_NAME>` in production.  
- **Scope:** Covers CI/CD pipeline execution, post‑deployment validation, monitoring, and rollback procedures.  
- **Audience:** AI Engineers, DevOps, Security, and Operations teams.

## 2️⃣ Prerequisites
| Item | Description | Inline reference |
|------|-------------|------------------|
| `Git` repository | Contains model artefacts and CI scripts | `` `repo-root/` `` |
| `Docker` / `Podman` | Container runtime for the inference service | `` `docker` `` |
| `kubectl` / `gcloud` | Cluster access for GCP (or `eks` for AWS) | `` `kubectl` `` |
| `OPA` policy bundle | Guardrail policies (see `runbook_config.yaml`) | `` `policies/` `` |
| `Model Registry` entry | Version `<MODEL_VERSION>` must exist | `` `model-registry/` `` |

## 3️⃣ Pre‑deployment Checklist
- [ ] Verify **model version** `v<MAJOR>.<MINOR>.<PATCH>` is approved in the Model Registry.  
- [ ] Confirm **performance threshold**: `$F1 \geq 0.85$` and `$latency \leq 100$ ms`.  
- [ ] Run **fairness audit**: ensure demographic parity difference ≤ `$0.10$`.  
- [ ] Ensure **data privacy** checks passed (PII masked, encryption at rest/in‑transit).  

## 4️⃣ Deployment Steps
```bash
# 4.1 Build Docker image
docker build -t $REGISTRY/<PROJECT>/<MODEL>:v<MAJOR>.<MINOR>.<PATCH> .

# 4.2 Push image
docker push $REGISTRY/<PROJECT>/<MODEL>:v<MAJOR>.<MINOR>.<PATCH>

# 4.3 Apply OPA guardrails (policy‑as‑code)
opa eval -i deployment.yaml -d policies/guardrail.yaml "data.policy.allow"

# 4.4 Deploy to Kubernetes (or Cloud Run)
kubectl apply -f k8s/deployment.yaml
kubectl apply -f k8s/service.yaml
```

## 5️⃣ Post‑deployment Validation
| Step | Command | Expected Result |
|------|---------|-----------------|
| **Smoke test** | `curl -X POST https://<ENDPOINT>/predict -d '{"feature":[0.5]}'` | HTTP 200 & JSON response within `$100$ ms` |
| **Model health** | `curl https://<ENDPOINT>/health` | `"status":"OK"` |
| **Performance test** | `python tests/perf_test.py --model v<MAJOR>.<MINOR>.<PATCH>` | F1 ≥ `$0.85$`, latency ≤ `$100$ ms` |

## 6️⃣ Monitoring & Alerting
- **Metrics** to collect: `model_f1`, `request_latency`, `error_rate`, `drift_score`.  
- **Alert thresholds**:  
  - `drift_score > $0.15$` → PagerDuty incident.  
  - `error_rate > 1%` for 5 min → Slack alert.  

## 7️⃣ Rollback Procedure
1. Identify previous stable image tag `v<MAJOR>.<MINOR>.<PATCH>-stable`.  
2. Run: `kubectl rollout undo deployment/<DEPLOYMENT> --to-revision=<REVISION>`  
3. Verify rollback with smoke test (step 5).
