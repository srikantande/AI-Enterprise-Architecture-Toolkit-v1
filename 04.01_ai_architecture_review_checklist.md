# AI Architecture Review Checklist

Use this checklist during every AI project architecture review:

## Pre-Architecture

- [ ] Business case approved with measurable OKRs
- [ ] Use case scored (reversibility, privacy, safety, fairness)
- [ ] Data assessment completed (availability, quality, sensitivity)
- [ ] Target state architecture defined (TOGAF Phase C/D)

## Security

- [ ] Data encryption at rest (CMEK) and in transit (TLS 1.3)
- [ ] VPC Service Controls / Endpoints configured
- [ ] IAM roles follow least privilege with audit logging
- [ ] Secret management via Secret Manager / Secrets Manager
- [ ] DLP scan on training data and prompts

## Architecture Quality

- [ ] DR strategy defined (RPO/RTO for AI workloads)
- [ ] Cost model created and approved
- [ ] Integration patterns documented (event-driven, API-led)
- [ ] Observability strategy (metrics, traces, logs, dashboards)
- [ ] Model versioning and lineage tracking enabled

## Compliance

- [ ] Regulatory impact assessment (GDPR, EU AI Act, industry-specific)
- [ ] Model Card completed
- [ ] Bias/fairness tests executed and documented
- [ ] Human-in-the-loop controls deployed *(if Tier 3)*
